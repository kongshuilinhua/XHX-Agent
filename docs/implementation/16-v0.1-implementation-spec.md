# v0.1 实施规格

v0.1 的目标是做出最小可运行 Agent Runtime：能初始化项目、加载模型配置、读取和搜索文件、通过 `apply_patch` 修改、推断验证命令、生成 Markdown 总结。

## 成功定义

v0.1 完成时，开发者可以在一个小型 Python 或 JavaScript/TypeScript 仓库中运行：

```bash
xhx init
xhx run "fix the failing test"
```

系统应能：

- 扫描项目并生成 `XHX.md`。
- 读取和搜索相关文件。
- 生成结构化补丁。
- 通过 `apply_patch` 修改文件。
- 推断验证命令。
- 在用户确认后运行验证。
- 输出 Markdown 总结。

## 包和入口

使用 uv：

```bash
uv init
uv add typer rich httpx pydantic langgraph
uv add --dev pytest
```

`pyproject.toml` 必须提供 CLI：

```toml
[project.scripts]
xhx = "xhx_agent.cli.main:app"
```

v0.1 必须支持：

```bash
xhx init
xhx chat
xhx run "task"
xhx run "task" --json
xhx config list
xhx config set-profile default
```

`--json` 在 v0.1 可以只输出 `RunResult`，不要求完整 headless RPC。

## CLI 行为

### xhx init

行为：

- 创建 `.xhx/` 目录。
- 创建 `.xhx/config.json`。
- 创建 `.xhx/profiles.json` 示例。
- 扫描项目并生成 `XHX.md`。

不覆盖规则：

- 如果文件已存在，默认不覆盖。
- 如果需要刷新，后续版本再提供 `--force`。

成功输出：

```text
Initialized xhx-agent project.
Created .xhx/config.json
Created .xhx/profiles.json
Created XHX.md
```

### xhx chat

行为：

- 启动简单 REPL。
- 支持输入普通任务文本。
- 支持 `/exit` 退出。

v0.1 不要求完整 TUI。

### xhx run "task"

行为：

- 创建 run id。
- 加载 config 和 profile。
- 运行线性 Agent 流程。
- 输出最终摘要路径。

默认流程：

```text
load_session
scan_project_if_needed
build_context
plan
gather
patch
verify
summarize
```

## 配置文件 schema

`.xhx/config.json`：

```json
{
  "version": 1,
  "default_profile": "mock",
  "workspace_root": ".",
  "max_file_bytes": 200000,
  "default_language_targets": ["python", "javascript", "typescript"],
  "write_policy": "apply_patch_only"
}
```

`.xhx/profiles.json`：

```json
{
  "profiles": [
    {
      "name": "mock",
      "provider": "mock",
      "base_url": "",
      "api_key_env": "",
      "model": "mock",
      "temperature": 0,
      "stream": false
    },
    {
      "name": "default",
      "provider": "openai-compatible",
      "base_url": "https://api.example.com/v1",
      "api_key_env": "XHX_API_KEY",
      "model": "qwen-plus",
      "temperature": 0.2,
      "stream": true
    }
  ]
}
```

规则：

- `api_key_env` 不存在时，真实模型调用必须失败并给出明确错误。
- v0.1 默认使用 `mock` profile，保证本地 fixture 和基础闭环可离线验证。
- 真实模型通过 `default` 或用户自定义 OpenAI-compatible profile 显式启用。

## 目录和文件

```text

  cli/
  runtime/
```

v0.1 可以先保留空包：

```text

```

空包必须有 README 或 module docstring 说明后续版本用途，避免误以为已实现。

## 模型调用

v0.1 提供两个 client：

- `OpenAICompatibleClient`
- `MockModelClient`

`MockModelClient` 用于单元测试和集成测试，必须能返回：

- 普通文本。
- tool call。
- patch 建议。

真实模型错误必须转为结构化错误：

```json
{
  "type": "model_error",
  "message": "Missing API key env XHX_API_KEY"
}
```

## 工具协议

v0.1 工具：

### read_file

参数：

```json
{
  "path": "README.md",
  "start_line": 1,
  "max_lines": 200
}
```

规则：

- 路径必须在 workspace 内。
- 默认按 UTF-8 读取。
- 超过 `max_file_bytes` 时返回失败摘要。

### search

参数：

```json
{
  "query": "def add",
  "glob": "*.py",
  "max_results": 50
}
```

规则：

- 优先使用 `rg`。
- 没有 `rg` 时使用 Python fallback。
- 返回文件、行号、片段。

### terminal

参数：

```json
{
  "command": "python -m pytest",
  "timeout_seconds": 120
}
```

规则：

- 必须经过 Safety。
- v0.1 safe 命令可执行。
- confirm 命令需要用户确认。
- deny 命令拒绝。

### apply_patch

参数：

```json
{
  "patch": "*** Begin Patch\n*** Update File: src/foo.py\n...\n*** End Patch"
}
```

规则：

- 只接受结构化 patch。
- 只允许 workspace 内相对路径。
- 输出 changed files。
- patch 失败时不继续验证。

## 线性 Agent Graph

v0.1 使用简单 LangGraph 状态流。

状态字段：

```json
{
  "run_id": "run_001",
  "task": "fix failing tests",
  "mode": "linear-edit",
  "messages": [],
  "plan": [],
  "tool_results": [],
  "changed_files": [],
  "verification_plan": null,
  "verification_result": null,
  "errors": []
}
```

节点：

- `classify`：判断 direct、research-only 或 linear-edit。
- `build_context`：读取 `XHX.md` 和基础项目摘要。
- `plan`：生成短计划。
- `gather`：读文件和搜索。
- `patch`：应用 patch。
- `verify`：推断并执行验证。
- `summarize`：生成 Markdown 总结。

## 项目扫描和 XHX.md

扫描内容：

- 当前目录名。
- 文件数量摘要。
- 识别语言。
- Python 配置：`pyproject.toml`、`pytest.ini`、`requirements.txt`、`tests/`。
- Node 配置：`package.json`、`tsconfig.json`。
- 可用脚本：`npm test`、`npm run build`、`npm run typecheck`。

`XHX.md` 最小模板：

```md
# XHX Project Map

## Project

- root: .
- detected languages:

## Structure

## Verification

## Notes
```

## 验证推断

Python：

1. 如果存在 `tests/`，推荐 `python -m pytest`。
2. 如果存在 `pytest.ini` 或 `pyproject.toml` pytest 配置，推荐 `python -m pytest`。
3. 如果无法判断，跳过并说明原因。

Node：

1. 如果 `package.json` 有 `test`，推荐 `npm test`。
2. 否则如果有 `typecheck`，推荐 `npm run typecheck`。
3. 否则如果有 `build`，推荐 `npm run build`。
4. 否则跳过并说明原因。

验证执行仍需要 Safety 判断。

## Markdown 总结模板

路径：

```text
.xhx/logbook/<run-id>.md
```

模板：

```md
# xhx-agent Run Report

## Task

## Plan

## Changed Files

## Commands

## Verification

## Evidence Summary

## Risks
```

v0.1 的 Evidence Summary 可以先来自工具结果摘要，不要求完整 Evidence Runtime。

## v0.1 测试

单元测试：

- config 加载。
- profile 加载。
- missing API key 错误。
- command risk classify。
- read_file path guard。
- search fallback。
- apply_patch 成功和失败。
- verification inference。
- Markdown report rendering。

集成测试：

- Python fixture：函数 bug + pytest。
- Node fixture：简单 build/test。
- deny command 被拒绝。
- patch 失败后不执行验证。

验收命令：

```bash
uv run pytest
xhx init
xhx run "analyze this repo" --json
```

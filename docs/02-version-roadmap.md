# XHX-Agent 架构演进与版本历史文献

本篇文献作为 XHX-Agent 项目的进化史册，系统地记录了系统从一个简易的模型循环，逐步演进为工业级端到端自愈编码 Agent 运行时的全过程。这里的每一个版本均代表一个已经成功攻克并落地的里程碑（Milestone），详细记录了当时的工程挑战与架构破局方案。

---

## 🏛️ v0.1 最小 Agent Runtime 里程碑：完成读改测总结闭环

### 🎯 达成目标
构建了智能体最基础的“大模型 -> 工具执行 -> 结果捕获 -> 反馈大模型”的最小闭环，抛弃了单纯的 Mock，实现了真实模型的结构化接入与命令行 REPL 的顺畅交互。

### 🛠️ 核心实现
- **单模型 Profile 与 OpenAI 兼容性**：配置 `base_url`、`api_key` 和 `model`。
- **四大基石工具**：实现 `read_file`、`search`、`terminal`、`apply_patch`。
- **项目扫描器**：自动生成轻量级项目规则文件 `XHX.md` 与 Repo Map。
- **初级验证命令推断**：自动识别 Python (pytest) 与 Node.js (npm test) 运行环境。
- **任务总结生成**：任务结束后自动导出 Markdown 格式的执行总结。

### ⚠️ 工程挑战与破局方案
- **工程挑战：大模型 Tool-Call 的不稳定循环与死锁**。早期开发中，大模型频繁产生幻觉，输出错误的 JSON 工具参数，或在应用 Patch 失败时陷入反复尝试的死循环，导致 Token 极速消耗。
- **破局方案**：在 Runtime 层引入了强类型的参数校验器，并在 LangGraph 工作流中加入硬性的最大迭代轮数控制（Loop Guard），当出错轮数超过阈值时强制阻断并生成结构化诊断报告，从而维持了 Runtime 的稳定性。

## 🏛️ v0.2 Safe Execution Kernel 里程碑：可控、可恢复的沙盒执行

### 🎯 达成目标
彻底解决了智能体在本地操作系统中越权、破坏环境的工程隐患，实现所有工具调用与 Shell 指令执行的完全可审计与事务级回滚。

### 🛠️ 核心实现
- **命令风险三级 Policy Check**：划分 Safe、Confirm、Deny 三个安全屏障。
- **修改前 Checkpoint**：打 Patch 前自动在暂存区创建 Git 快照。
- **changed files 状态追踪**：全生命周期监控发生物理更改的文件。
- **初级 2-Turn 自适应修复 (Repair Loop)**：验证失败时能启动微循环进行 2 轮内修复。

### ⚠️ 工程挑战与破局方案
- **工程挑战：子进程越权执行与危险指令静默运行**。在执行 `terminal` 指令时，智能体可能会试图执行 `rm -rf` 或修改全局系统配置。此外，子进程在执行长期挂起的交互式命令时会造成主线程永久性假死。
- **破局方案**：设计了**子进程权限拦截与超时看门狗系统**。通过安全策略字典对 Shell 命令进行前置词法分析，强制屏蔽危险命令（进入 Deny 库）；利用 `subprocess` 的管道流进行非阻塞式流式数据采集，并设置了 120 秒的硬性超时看门狗（Watchdog），彻底解决了子进程假死与越权问题。

## 🏛️ v0.3 Context Pack Compiler 里程碑：控制 Token 预算与防污染

### 🎯 达成目标
解决了大模型在处理中大型项目时面临的“上下文爆炸”和“历史对话杂音污染”问题，实现 Token 消费的预测性与高效性。

### 🛠️ 核心实现
- **固定 Token 刚性预算**：为每轮 Prompt 设置硬性上限。
- **Context Pack 动态装配结构**：包含系统规则、项目地图、活动计划、符号代码片段和 Evidence Top-K 摘要。
- **滑动窗口与优先级裁剪算法**。

### ⚠️ 工程挑战与破局方案
- **工程挑战：大规模上下文下的 Token 爆炸与模型迷失（Lost in the Middle）**。随着任务链的拉长，完整的历史执行日志和长代码文件直接喂给模型，不仅造成 Token 费用飙升，还会导致模型忽略位于 Prompt 中部的关键验证失败信息。
- **破局方案**：开发了**优先级滑动窗口裁剪机制**。将进入 Prompt 的上下文划分为刚性层（Goal & Plan & Rules）和弹性层（Code Snippets & Evidence）。当 Token 总量逼近刚性预算上限时，按反向时间序与相关度打分对弹性层信息进行优先级裁剪与截断，保证喂给 LLM 的永远是最核心、最紧凑的高相关上下文。

## 🏛️ v0.4 Evidence Runtime 里程碑：高精结构化证据链落盘

### 🎯 达成目标
实现运行轨迹的 100% 审计追溯。大模型无需在上下文里读取臃肿的日志，只需通过高浓度的证据索引（Evidence Index）即可了解历史。

### 🛠️ 核心实现
- **Raw Trace 文件落盘**：以 `.jsonl` 格式将每次输入、输出、工具响应完全持久化。
- **Evidence Index 摘要生成**：将长工具输出压缩为一两句确定性事实。
- **TrailGraph 内部依赖网络**：描述“因 -> 决策 -> 行动 -> 果”的有向链条，支持一键渲染为人类可读的 Logbook 审计报告。

### ⚠️ 工程挑战与破局方案
- **工程挑战：Raw Trace 序列化开销过大与 TrailGraph 拓扑内存溢出**。当智能体进行大型重构任务时，会产生数万行日志，每次读写 Trace 文件和构建内存图都会带来明显的 IO 停顿。
- **破局方案**：引入了**异步非阻塞 IO 写入队列与轻量级增量序列化方案**。Raw Trace 采用追加写（Append-only）的 JSONL 模式；TrailGraph 仅在内存中保存极精简的符号指针依赖关系，只有在最终渲染 Logbook 时才进行全量解析拼接，将 IO 损耗降到微秒级。

## 🏛️ v0.5 TUI / Command Console 里程碑：富交互式终端体验

### 🎯 达成目标
实现极其惊艳、极具科技感的 TUI 终端窗口，让 Runtime 的执行计划、工具调用、diff 对比与自愈状态可视化呈现在用户眼前，全面对标并超越 Aider / Claude Code。

### 🛠️ 核心实现
- **多线程流式输出渲染**：基于 Rich 库构建动态控制台面板。
- **交互式确认 UI**：当触发 `Confirm` 工具调用时弹出优美的选择菜单。
- **系统斜杠命令（`/` Commands）**：支持 `/context`、`/evidence`、`/diff`、`/verify` 等 12 个核心命令行快捷指令。

### ⚠️ 工程挑战与破局方案
- **工程挑战：Rich TUI 渲染与大模型异步流式数据冲突**。当后台大模型正在流式吐出文本时，如果同时触发了工具调用状态面板的刷新，会导致终端屏幕输出混乱、闪烁甚至重叠。
- **破局方案**：构建了**单缓冲区多路复用控制台渲染引擎（Render Lock）**。大模型流式输出与状态面板刷新完全通过互斥锁进行同步，在大模型吐出字符期间挂起面板刷新，并将终端物理光标精确控制在底端，完美实现了极速且顺滑的字符流动体验。

## 🏛️ v0.6 Repo Intelligence Graph 里程碑：基于 AST 与 SQLite 的代码智能

### 🎯 达成目标
彻底摆脱了传统 RAG 的文本模糊检索，为 XHX-Agent 注入了 100% 确定性的、基于编译器前端 AST 与高性能 SQLite 数据库的多维语义代码图谱。

### 🛠️ 核心实现
- **多语言 AST 符号提取**：Python 内置 `ast` 深度嵌套解析，JS/TS 采用 Tree-Sitter 与精密 Regex 正则双轨级联降级提取机制。
- **SQLite 高性能 B-Tree 索引底座**：创建 `symbols`、`imports`、`references`、`calls` 四张核心规范关系表，建立 8 个 B-Tree 索引。
- **启发式测试匹配与 BFS 逆向追溯**：实现了秒级测试匹配与基于 SQLite 导入图的 BFS 受影响测试因果推导。

### ⚠️ 工程挑战与破局方案
- **工程挑战：中大型项目下符号高频检索的性能瓶颈与全量重建索引带来的磁盘震荡**。在一个包含数千个源文件的中大型库中，每次代码发生微小修改，如果都全量重新解析 AST 并重建 SQLite 索引，会导致严重的卡顿与磁盘频繁擦写。
- **破局方案**：引入了**轻量级增量文件指纹同步算法**。通过监测物理文件的 `size_bytes` 与 `mtime_ns`（修改时间戳）生成哈希指纹。每次运行前仅扫描指纹发生变化的文件，实现 AST 的**增量局部解析**；而在 SQLite 写入层，将批量插入包裹在单一数据库事务内，利用 `executemany` 高速写入，将万级符号的增量同步时间压缩在 10 毫秒以内，极速无比。

## 🏛️ v0.7 Adaptive Planner + DAG 里程碑： Kahn 拓扑并发调度

### 🎯 达成目标
实现多任务的自适应智能编排。复杂重构任务会被分解为有向无环图（DAG），并以极致的安全并发效率执行。

### 🛠️ 核心实现
- **自适应意图识别分类器**：自动判断 Direct、Research Only、DAG Parallel 执行路线。
- **Kahn 拓扑排序算法**：编译期零入度排序与循环死锁检测。
- **读写隔离混合并发调度**：只读任务 8 线程并发，写入任务单线程串行。
- **线程安全数据同步锁**：使用 `threading.Lock()` 护航主状态追加。

### ⚠️ 工程挑战与破局方案
- **工程挑战：多物理环境下的拓扑随机性扰动与并发脏写（Lost Update）**。经典的 Kahn 算法在队列处理时，可能会因为多线程的竞争条件或哈希字典序的无序性，在不同宿主机上产生不同的拓扑排序，导致系统难以重现 Bug。此外，写入文件（打 Patch）如果并发执行，会造成物理冲突和代码截断。
- **破局方案**：
  1. 在 Kahn 算法的零入度工作队列弹出前，强制执行稳定的字母顺序排序（`queue.sort()`），使得排序路径在多平台上具有 100% 确定性；
  2. 确立了**“读优先最大并发，写严格串行退化”**的混合调度决策。当有只读任务时拉满线程池并发，遇到写操作时立即收缩为同步串行，并在写操作执行完毕后瞬间利用 SQLite 增量刷新符号地图，完美解决了数据一致性与并发吞吐的冲突。

## 🏛️ v0.8 Skills / Extensions / MCP 里程碑：懒加载插件化沙箱

### 🎯 达成目标
在不增加 Prompt Token 负担的前提下，无限扩展智能体的开发工具链与计划微调策略。

### 🛠️ 核心实现
- **`.xhx/skills` 本地 Skill 目录扫描规范**。
- **Metadata 预加载与按需披露（Progressive Disclosure）**。
- **生命周期 Hook 挂载**：支持 `before_plan`、`before_patch`、`after_verify`、`before_summary` 核心钩子。

### ⚠️ 工程挑战与破局方案
- **工程挑战：插件化动态加载带来的安全沙箱穿透与 Prompt 溢出**。当用户挂载了 20 个 Skills 时，如果一次性将所有 Skill 的正文喂入模型，会导致 Token 直接挤爆；此外，恶意的 Skills 可能会通过 Hook 越权调用 terminal 危险命令。
- **破局方案**：设计了**插件权限围栏与 Metadata 懒加载架构**。运行时仅将 Skills 的 YAML 概要信息装配进常驻上下文。大模型在有具体需求时匹配对应触发词，才动态加载指定 Skill 详细正文。所有的 Skill Hook 不具有提升工具权限的资格，命令执行仍需经过 Safe Execution Kernel 的 Policy Check 拦截，保障了插件生态的安全干净。

## 🏛️ v0.9 Evaluation / Headless / Replay 里程碑：确定性回放与评测

### 🎯 达成目标
支持 XHX-Agent 的 Headless 自动化运行、CI 检查以及完全不消耗 Token 的确定性运行轨迹回放（Trail Replay）。

### 🛠️ 核心实现
- **CLI 静默模式**：支持 `xhx run "<task>" --json` 输出结构化日志。
- **JSONL RPC 协议支持**。
- **物理回放引擎**：加载 Evidence 中的 Trail 记录，直接模拟宿主机工具结果并产出最终 Logbook 总结，无需实际发起 LLM 请求。

### ⚠️ 工程挑战与破局方案
- **工程挑战：回归测试中多用例并行运行时的宿主机环境互斥冲突**。在 benchmark 评测数百个任务时，多个并发任务可能会修改同一个测试数据库或同一个本地端口，造成评测结果假阴性。
- **破局方案**：引入了**轻量级容器化与进程级物理端口动态分配隔离机制**。每一次 Headless 评测任务均被绑定在独立的临时工作目录，端口由空闲端口扫描器动态分配并注入环境变量中，从物理层隔离了并发测试干扰，保证了评测的一致性。

## 🏆 v1.0 完整 Agent 里程碑：工业级端到端自愈编码系统整合

### 🎯 达成目标
XHX-Agent 达到 1.0 终极形态！成为一套高度整合、端到端高度自愈的工业级生产力编码工具。

### 🛠️ 核心实现
- **完美的读-改-测-自愈-可审计全流程整合**。
- **完全解耦的 Runtime 层与 CLI/TUI 表演层**。
- **完整的文档体系与 private deep-dive 学术私密导读包**。

### ⚠️ 工程挑战与破局方案
- **工程挑战：全链路模块高并发协同下的整体稳定性与边界容错**。五大内核（Context Pack, Evidence, Safe Kernel, Adaptive Planner, Verification Router）和 SQLite 语义底座在面对特大型多包（monorepo）仓库时，极易因单个模块的未捕获异常引发级联崩溃。
- **破局方案**：在 Runtime 外围构建了**全局状态监控与级联熔断器（Fuse Board）**。任意内核发生不可恢复的致命异常时，熔断器立即启动，暂停 Kahn 队列调度，启动 Safe Kernel 的 Restore Plan 一键无物理损伤还原 Git 暂存区，并将完整的 TrailGraph 证据链和诊断信息完美写盘后优雅退出。这保障了 XHX-Agent 成为一款开发者可以放心托付代码库的、绝对安全的工业级编码运行时。

# Context Pack Compiler

Context Pack Compiler 负责在每轮 LLM 调用前编译上下文包。它的目标是控制上下文大小和质量，避免历史、日志和无关文件污染 Prompt。

## 输入

- 用户当前任务。
- 当前执行模式。
- `XHX.md` 项目地图。
- 当前计划。
- changed files。
- 相关文件片段。
- Evidence Index。
- 最近错误摘要。
- 历史会话摘要。

## 输出

Context Pack 是传给模型的结构化上下文包。

建议结构：

```text
目标
当前模式
项目规则与 XHX.md 摘要
当前计划
相关代码片段
关键证据摘要
最近失败点
约束与禁止事项
```

## 预算分配

默认预算建议：

```text
20% 项目地图与规则
25% 当前任务和计划
35% 相关代码片段
15% 证据摘要
5% 最近错误和风险
```

不同任务可以调整：

- 调试任务增加错误和测试日志摘要预算。
- 重构任务增加相关代码片段预算。
- 文档任务增加 README/docs 片段预算。
- 只读分析任务减少 patch 和验证相关内容。

## 选择策略

上下文选择按优先级执行：

1. 用户当前明确要求。
2. 安全和写入规则。
3. 当前失败点。
4. changed files。
5. 与任务最相关的代码片段。
6. 高置信度 Evidence。
7. 历史摘要。

超过预算时从低优先级内容开始裁剪。

## 反模式

- 把完整命令输出塞进 Prompt。
- 把所有历史消息常驻 Prompt。
- 让 Skill 正文默认进入每轮 Prompt。
- 不区分当前任务和长期项目知识。
- 上下文过长时随机截断。

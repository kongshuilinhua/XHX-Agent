# 步骤树（Block ②c）实现计划

> **For agentic workers:** 逐 Task、TDD、频繁提交。**硬约束：turn/model/耗时/token 全真实，禁 hardcode/mock。** 步骤用 `- [ ]`。

**Goal:** 把每轮模型调用渲染成 `── turn N · {model} · {秒}s · in {prompt}/out {completion}` 的轮次树头，其下工具调用缩进嵌套，形成可读的执行步骤树。

**Architecture:** 直接长在 ②b 上：给 `token_usage` 事件补真实 `turn`（`chat_and_count` 加 turn 参数，7 处调用传入），时间线把 ②b 的指标行升级为轮次树头；工具行本就缩进，天然成树。**不动模型 provider 包、不改编排流程。** 深层子 agent 嵌套本块不做（YAGNI）。

**Tech Stack:** Python 3.12 + Textual/Rich；pytest。

### Task 2: 时间线轮次树头 + 配色

**Files:** Modify `src/xhx_agent/tui/textual_app.py`（`_timeline_line_for_event` 的 token_usage 分支）、`src/xhx_agent/tui/format.py`（`line_style`）

**行为：**
- token_usage 分支（②b 已建）改为：
  ```
  model = p.get("model")
  if not model:
      return None
  secs = p.get("duration_ms", 0) / 1000
  prompt_t = human_tokens(int(p.get("prompt", 0) or 0))
  comp_t = human_tokens(int(p.get("completion", 0) or 0))
  turn = int(p.get("turn", 0) or 0)
  if turn > 0:
      return f"── turn {turn} · {model} · {secs:.1f}s · in {prompt_t}/out {comp_t}"   # 轮次树头
  return f"  · {model} · {secs:.1f}s · in {prompt_t}/out {comp_t}"                      # 后台/子 agent 回退（②b 行为）
  ```

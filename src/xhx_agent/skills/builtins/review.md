## 代码审查清单

当用户要求审查代码时，请按以下维度检查：

### 1. 正确性
- 逻辑是否正确？边界条件处理了吗？
- 空值/nil 检查是否充分？
- 循环终止条件是否正确？

### 2. 安全性
- 是否有 SQL 注入、命令注入风险？
- 用户输入是否做了验证和清理？
- 敏感信息（密钥、密码）是否硬编码？

### 3. 性能
- 是否有不必要的内存分配？
- 是否有 O(n²) 可以优化为 O(n) 的地方？
- 文件 I/O 是否可以合并？

### 4. 可维护性
- 命名是否清晰？
- 函数是否过长（超过 50 行）？
- 是否有重复代码可以提取？

### 5. 测试
- 是否有对应的测试用例？
- 是否覆盖了主要异常路径？

审查完成后给出：
1. 严重问题（必须修复）
2. 建议改进（建议采纳）
3. 总体评价
